## Directory Purposes

**`kimun/`:**
- Purpose: Django project configuration layer
- Contains: Settings, root URLs, WSGI
- Key files: `settings.py`, `urls.py`

**`usuarios/`:**
- Purpose: Identity, access control, audit, notifications
- Contains: Custom user model, auth views, decorators, middleware, signals, email utils
- Key files: `models.py`, `decorators.py`, `signals.py`, `middleware.py`

**`cursos/`:**
- Purpose: Core learning content and enrollment tracking
- Contains: Course, lesson, material, category models and CRUD views
- Key files: `models.py`, `views.py`, `forms.py`, `admin.py`

**`evaluaciones/`:**
- Purpose: Assessment engine with question banks and attempt tracking
- Contains: Evaluation, question, alternative, attempt models
- Key files: `models.py`, `views.py`, `forms.py`

**`certificados/`:**
- Purpose: Certificate generation and workflow management
- Contains: Certificate model with PDF generation via WeasyPrint
- Key files: `models.py`, `views.py`

**`tareas/`:**
- Purpose: Assignment submissions and grading
- Contains: Task and submission models
- Key files: `models.py`, `views.py`, `forms.py`

**`calendario/`:**
- Purpose: Calendar events with iCal export
- Contains: Event model and calendar views
- Key files: `models.py`, `views.py`

**`anuncios/`:**
- Purpose: Internal communications with read tracking
- Contains: Announcement and read receipt models
- Key files: `models.py`, `views.py`, `forms.py`

**`reportes/`:**
- Purpose: Read-only analytics and dashboards
- Contains: Views only (no models)
- Key files: `views.py`

**`templates/`:**
- Purpose: All HTML templates organized by app
- Contains: Base layout, per-app template directories, partials
- Key files: `base.html`, `inicio.html`

**`static/`:**
- Purpose: Source static assets before collection
- Contains: Custom CSS, JS, images
- Key files: `css/kimun.css`, `js/toast.js`, `img/alumco.png`

**`staticfiles/`:**
- Purpose: Django `collectstatic` output
- Contains: All collected static files including admin assets
- Generated: Yes
- Committed: Yes (in this repo)

**`tools/`:**
- Purpose: One-off maintenance and migration scripts
- Contains: Various fix/rewrite scripts
- Generated: No
- Committed: Yes

## Key File Locations

**Entry Points:**
- `kimun/wsgi.py`: WSGI application entry
- `manage.py`: Django CLI entry
- `kimun/urls.py`: Root URL dispatcher

**Configuration:**
- `kimun/settings.py`: All Django settings, database config, CKEditor config, storage config
- `.env`: Environment variables (SUPABASE credentials, secret key)
- `.env.example`: Template for required env vars

**Core Logic:**
- `usuarios/decorators.py`: Authorization system
- `usuarios/signals.py`: Audit logging and side effects
- `usuarios/middleware.py`: IP address capture
- `cursos/utils.py`: Progress calculation utility
- `usuarios/utils.py`: Email notification utilities

**Testing:**
- `test_visuals.py`: Visual regression tests
- `usuarios/tests.py`: Not present (no tests written)
- `cursos/tests.py`: Not present (no tests written)
- `evaluaciones/tests.py`: Not present (no tests written)

## Naming Conventions

**Files:**
- Models: `models.py` (standard Django)
- Views: `views.py` (standard Django)
- URLs: `urls.py` (standard Django)
- Forms: `forms.py` (standard Django)
- Admin: `admin.py` (standard Django)
- Utils: `utils.py`
- Signals: `signals.py`
- Middleware: `middleware.py`
- Decorators: `decorators.py`

**Directories:**
- Apps: lowercase plural Spanish names (`usuarios`, `cursos`, `evaluaciones`, `certificados`, `tareas`, `calendario`, `anuncios`, `reportes`)
- Templates: per-app subdirectories under `templates/`
- Static: per-type subdirectories (`css/`, `js/`, `img/`)

## Where to Add New Code

**New Feature (e.g., new entity):**
- Primary code: Create a new Django app directory at project root
- Models: `newapp/models.py`
- Views: `newapp/views.py`
- URLs: `newapp/urls.py`
- Forms: `newapp/forms.py`
- Register app in `kimun/settings.py` under `INSTALLED_APPS`
- Wire URLs in `kimun/urls.py` with `path('newapp/', include('newapp.urls'))`
- Templates: `templates/newapp/`

**New Template:**
- Place in `templates/<app_name>/<template_name>.html`
- Extend `templates/base.html`

**New Static Asset:**
- Source file: `static/css/`, `static/js/`, or `static/img/`
- Run `python manage.py collectstatic` to update `staticfiles/`

**New Utility Function:**
- If app-specific: `cursos/utils.py` or `usuarios/utils.py`
- If cross-cutting: Create a new `common/` app or place in `usuarios/utils.py`

**New Decorator:**
- Add to `usuarios/decorators.py`
- Import from there in other apps' views

**New Signal:**
- Add receiver to `usuarios/signals.py` (or app-specific `signals.py` if created)
- Ensure app config `ready()` method imports signals (see `usuarios/apps.py`)

**New Management Command:**
- Create `newapp/management/commands/<command_name>.py`

## Special Directories

**`staticfiles/`:**
- Purpose: Collected static files for production serving
- Generated: Yes (by `collectstatic`)
- Committed: Yes (present in repository)

**`media/`:**
- Purpose: User-uploaded files in development
- Generated: Yes (by file uploads)
- Committed: No (empty or `.gitkeep`)
- Production: Files stored in Supabase Storage via `django_supabase_storage`

**`tools/`:**
- Purpose: One-off scripts for fixing data or rewriting templates
- Generated: No
- Committed: Yes
